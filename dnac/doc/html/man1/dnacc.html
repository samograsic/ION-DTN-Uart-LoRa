NAME

dnacc - child agent for DTN node auto-configuration

SYNOPSIS

dnacc port_nbr password_path_name

DESCRIPTION

dnacc is a utility program that runs as an application in an ION node that has been automatically configured by the dnac utility running in a separate "sponsor" node on the same computer. Its purpose is to generate the new node's public/private key pair and pass the public key back to the sponsor node, which will be listening for that information on the TCP socket identified by port_nbr. This enables the sponsor node to attest to the new node's authenticity by asserting its public key to the network's delay-tolerant public key infrastructure.

EXIT STATUS

"0"

dnacc has terminated.

FILES

The file identified by password_path_name is expected to contain the passphrase that will enable the newly computed private key to be stored in encrypted representation. The file MUST be accessible only to the owner of the dnacc executable.

ENVIRONMENT

No environment variables apply.

DIAGNOSTICS

The following diagnostics may be issued to the ion.log log file:

dnacc can't attach to DTKA system.

dtka has not yet been initialized.
